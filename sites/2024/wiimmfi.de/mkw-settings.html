MKW specific settings of user »Gemdation«

Important UDP port notes

  • You can define a specific UDP port for NATNEG, MASTER and peer-to-peer communication.
  • When the UPnP request (done by CTGP or an updated Wiimmfi patcher) was successful, the configured UDP port will be ignored.
  • To change the port, click the table member and insert:
    • Value »0« to disable this feature.
    • Value »a« (auto) set the default port based on the serial number of the console.
    • A value between 16384 and 65535 to define a specific port.
For more details, check out this forum thread.

Important payload notes

  • Here you can select the payload (game update) version for Mario Kart Wii only.
  • When playing CTGP these payload settings are ignored.
  • The primary setting is the payload branch: Here you decide between following the Public branch (which is the normal, non-test version everyone else gets too), or the Beta branch which is for the Beta tests.
  • Another setting is the payload wish: Here you can select a specific payload version. If the selected version is not (longer) available, the payload branch is used as fallback. Usually you can keep this setting as »— none —« unless otherwise instructed by the Wiimmfi team.
  • The payload is only loaded once after game start at the first login. If you want to switch between payloads, you have to restart the game.
  • Use both columns of latest download to verify your activities.
  • Cookies and JavaScript must be enabled to use this page.
